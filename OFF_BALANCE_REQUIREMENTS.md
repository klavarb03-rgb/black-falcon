# Позабалансовий облік - Вимоги

## 🎯 Проблема
Матеріальні цінності часто надходять без документів (накладних). Документи можуть прийти через тиждень-два або взагалі не прийти. Потрібен механізм обліку таких МЦ.

## 💡 Рішення

### 1. Два статуси обліку:
- **off_balance** - позабаланс (без документів)
- **on_balance** - на балансі (з документами)

### 2. Процес роботи:

#### А) Товар прийшов БЕЗ документів:
1. Реєструємо МЦ зі статусом `off_balance`
2. Заповнюємо всі дані крім документів
3. МЦ доступні для використання
4. В звітах показуємо окремо як "Позабаланс"

#### Б) Документи прийшли пізніше:
1. Знаходимо МЦ в позабалансі
2. Вводимо дані накладної
3. Переводимо на баланс (`on_balance`)
4. Створюємо операцію переводу

### 3. База даних:

```sql
-- Нові поля в items
ALTER TABLE items ADD COLUMN balance_status VARCHAR(20) DEFAULT 'off_balance';
ALTER TABLE items ADD COLUMN document_id UUID NULL;
ALTER TABLE items ADD COLUMN document_date DATE NULL;
ALTER TABLE items ADD COLUMN document_number VARCHAR(50) NULL;
ALTER TABLE items ADD COLUMN supplier_name VARCHAR(255) NULL;
```

### 4. UI компоненти:

#### Dashboard:
- Окремий лічильник "Позабаланс: X шт"
- Кольорове розділення

#### Items List:
- Фільтр "Баланс/Позабаланс/Всі"
- Іконки: 📄 (з документами), ⚠️ (без документів)
- Кнопка "Оформити документи" для позабалансових

#### Нова форма "Оформлення документів":
- Номер накладної
- Дата накладної
- Постачальник/Донор
- Кнопка "Перевести на баланс"

### 5. Звіти:
- Окремий розділ "Позабалансові МЦ"
- Звіт "Очікують документи"
- Історія переводу на баланс

### 6. Права доступу:
- Manager може додавати в позабаланс
- Leader/Admin можуть переводити на баланс
